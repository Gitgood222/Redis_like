# 项目描述

基于 C++20 实现的类 Redis 内存数据库，使用 RESP 协议，`redis-cli` 可直接连接。

## 架构概览

单线程事件循环（`select` 多路复用，Linux 下预留 epoll），命令路由采用 `unordered_map` 做 O(1) 分发。支持 RDB 快照 + AOF 日志两种持久化方式，以及惰性过期 + 定期随机采样的键淘汰策略。

## 5 种基本数据类型的实现

| 类型 | C++ 底层 | 编码标记 | 是否手写数据结构 |
|------|---------|---------|----------------|
| **String** | `std::string` | `kRaw` | 否 |
| **Hash** | `std::unordered_map<string, string>` | `kHT` | 否 |
| **List** | `std::deque<string>` | `kRaw` | 否 |
| **Set** | `std::unordered_set<string>` | `kHT` | 否 |
| **ZSet** | 自定义 `SkipList` | `kSkipList` | **是** |

### String — `std::string`

直接使用 C++ 标准库的 `std::string`，没有实现 SDS。SET 存值，GET 原样返回。INCR/DECR 通过 `stod` / `to_string` 做数字转换。编码始终为 `kRaw`。

### Hash — `std::unordered_map<string, string>`

field → value 的哈希表映射。HSET 写入键值对，HGET 按 field 查询，HGETALL 遍历整个 map。未实现 ziplist 编码——无论字段多少都用哈希表。

### List — `std::deque<string>`

双端队列。LPUSH/RPOP 操作头部，RPUSH/LPOP 操作尾部，均为 O(1)。LRANGE 支持负索引（从尾部倒数）。未实现 QuickList（ziplist + linkedlist 混合）。

### Set — `std::unordered_set<string>`

无序集合。SADD 插入，SREM 删除，SISMEMBER O(1) 判断，SMEMBERS 遍历返回，SPOP 随机弹出。未实现 IntSet 编码——即使全为整数也使用哈希表。

### ZSet — 自定义 `SkipList`（**唯一手写的数据结构**）

深度参考 Redis `zskiplist.c` 从零实现：

- **节点结构** `SkipListNode`：score（double）、member（string）、forward 数组（多级前向指针）、backward（level 0 后向指针）、span（跨度，O(logN) 排名计算）
- **最大层高** `kMaxLevel = 32`，与 Redis 一致
- **随机层高** 幂次分布（p=0.25）+ `std::mt19937`，与 Redis 算法一致
- **辅助索引** 内部维护 `unordered_map<string, double>`，使 ZSCORE 达到 O(1)
- **插入策略** 使用 update/rank 数组定位，已存在 member 先删再插（更新 score）

## 为什么只有 ZSet 手写了数据结构

String、Hash、List、Set 的需求可以被 STL 容器直接满足：

- 字符串不需要二进制安全（C++ 的 `std::string` 天然支持 `\0`）
- 哈希表就是 map/set 的 O(1) 查找
- 双端操作 `std::deque` 天然支持头尾 O(1)

ZSet 不同——它需要同时满足：**按 score 排序 + 按 rank 查排名 + 按 score 范围查找 + 按 member O(1) 查 score**。STL 里没有单个容器能同时胜任：`set` 只能按 key 排序无法按 rank 定位，`vector` 插入 O(N)。跳表是唯一能 O(logN) 覆盖全部这些操作的结构，所以必须自己写。

## 数据流

1. `main.cpp` → `RedisServer::Init()` 监听端口，注册到事件循环
2. `EventLoop::RunOnce()` 用 `select()` 多路复用
3. 客户端可读时 `OnClientEvent()` → `recv()` → `RespCodec::Feed()` 解析 RESP 协议
4. `CommandRouter::Execute()` 查 `unordered_map<string, CmdHandler>` 分发命令
5. 命令处理器操作 `Dict`（全局键空间），返回 RESP 编码响应
6. 响应写入 `write_buf`，非阻塞 `send()` 发回

| String/Key | SET, GET, DEL, EXISTS, EXPIRE, TTL, PEXPIRE, PTTL, TYPE |
| Hash | HSET, HGET, HDEL, HEXISTS, HKEYS, HVALS, HGETALL, HLEN, HSTRLEN |
| List | LPUSH, RPUSH, LPOP, RPOP, LLEN, LRANGE, LINDEX |
| Set | SADD, SREM, SISMEMBER, SMEMBERS, SCARD, SPOP |
| ZSet | ZADD, ZREM, ZSCORE, ZRANK, ZREVRANK, ZRANGE, ZREVRANGE, ZCARD |
| Server | PING, PONG, COMMAND (basic) |
